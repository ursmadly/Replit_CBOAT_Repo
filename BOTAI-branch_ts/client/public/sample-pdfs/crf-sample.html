CRF Completion Guidelines

Protocol: PRO001

Version: 1.2

Date: March 10, 2025

1. Introduction

This document provides guidance for the accurate and complete documentation of clinical data in the Case Report Forms (CRFs) for the PRO001 clinical trial. Proper completion of CRFs is essential for ensuring data quality, integrity, and compliance with regulatory requirements.

Important: All site staff responsible for data entry should review these guidelines thoroughly before initiating any data collection activities.

2. General Principles

2.1 Data Entry Standards

  • All data must be entered in English
  • Use black or blue ink for paper CRFs (if applicable)
  • Enter dates in the format DD-MMM-YYYY (e.g., 15-Mar-2025)
  • Enter times using the 24-hour clock format (e.g., 14:30 instead of 2:30 PM)
  • Do not leave fields blank – use appropriate codes for missing data

2.2 Missing Data Codes

Code Meaning When to Use
NK Not Known Information was sought but could not be obtained
ND Not Done Test or procedure was not performed
NA Not Applicable Question is not relevant based on other information

3. Electronic Data Capture (EDC) System

The trial uses an electronic data capture system for all CRF data. Access is controlled through individual user credentials. All actions in the system are tracked with an audit trail that captures the user, date, time, and nature of each change.

3.1 Login Procedures

  • Access the EDC system at [secure URL]
  • Enter your personal username and password
  • Passwords must be changed every 90 days
  • Never share login credentials with others

3.2 Data Entry Workflow

  1. Select the appropriate study in the dashboard
  2. Select the site number
  3. Select the patient ID
  4. Select the relevant CRF page
  5. Enter data according to the guidelines in this document
  6. Save the page when complete

4. Module-Specific Guidelines

4.1 Demographic Information

  • Enter the patient's initials as first, middle, last (e.g., JDR for John David Roe)
  • Record only month and year of birth (not the day) to protect patient privacy
  • Enter all demographic data at the Screening visit

4.2 Medical History

  • Enter all relevant medical conditions present within 5 years prior to screening
  • For each condition, specify whether it is ongoing or resolved
  • If resolved, provide the end date (month and year)

4.3 Laboratory Results

  • Enter all values exactly as reported by the laboratory
  • Include the units of measurement
  • Flag any values outside of reference ranges
  • For missing samples, provide a reason using the appropriate code

4.4 Adverse Events

  • Report all adverse events regardless of severity or relationship to study treatment
  • Provide onset date, resolution date (if applicable), severity, and relationship to study drug
  • For serious adverse events, complete the additional SAE form within 24 hours

5. Query Resolution Process

Data queries will be generated by the system or by data management personnel when data is missing, inconsistent, or appears incorrect. All queries should be addressed promptly.

5.1 Query Response Timeframes

  • Safety-related queries: within 48 hours
  • Non-safety-related queries: within 5 business days

5.2 Query Resolution Steps

  1. Review the query and investigate the issue
  2. If a correction is needed, update the data field
  3. Provide a comment explaining the resolution
  4. Submit the response

6. Quality Control

Regular quality checks should be performed by site personnel to ensure data accuracy and completeness before database lock.

Remember: The quality of clinical trial data directly impacts the validity of study conclusions. Accurate and complete CRF data is essential for patient safety and regulatory compliance.
